Inheritance

In this chapter we'll be addressing inheritance.

This is the true magic of object oriented programming.

Inheritance allows you to reuse you code and to centrally maintain your code.

I'll be using the same example as in the previous chapter, but a few minor changes might occur.


1.1 Concept

Let me bring you a little closer to the concept of Data Encapsulation.

Data Encapsulation means that instances created in a specific class can only call methods in the same class as the instance.

These methods can in turn only access attribute values of objects that are "stored" in the same class.

The attribute values of objects can only be "stored" if a constructor exists in the class.

Example: If you create an instance in class A, then you can only access the methods in class A.

If we would want to access the methods in Class B, then we would get an error.

Remember: Classes are generally isolated (except for Inheritance).


1.2 Forms

1.2.1 Single

Now back to our example from the previous chapter.

If we would want to have two separate classes called Dog and Cat with the same attributes and methods as in our Animal class, then the two classes Dog and Cat can inherit from the Animal class.

In the class Animal we define the attributes and methods that the two class have in common.

If you now want to add attribute(s) and method(s) that are specific to a subclass, then you can do that by defining them in the subclass.

Example:

Attributes: Number of legs, color of eyes and skin color.

Methods: get_n_o_legs, get_skin_col, get_color_eyes.

These exist in both of our subclasses (Dog, Cat) since we inherited from the Animal class.

If we now would want to add an attribute and a method that only applies to Dog such as color of collar & get_color_of_collar, then we can do this by defining it in the Dog class.

The same could be done for the Cat class. If a cat would have an attribute and a method that only applies to them, then we can do this by defining it in the Cat class.

This approach is called Generalization ⇨ Specialization.


1.2.2 Multi-Level

Same Situation as before.

We have a class called Animal and two subclasses called Dog and Cat.

This time I'll be using different attributes for the subclasses!

Dog inherits from the class called Animal and in the Dog class we add an attribute and a method called Weight & get_Weight.

If we now want to have the same attributes and methods from the Dog class in our Cat class, then we need to inherit from our Dog class.

The approach is called Multi-level, because the Cat class inherits from the Dog class and the Dog class inherits from the Animal class.


1.2.3 Multiple

I am sorry guys! I couldn't think of an example with multiple inheritance for our current example.

Here's the new example:

Imagine that we have two classes called Grades and Students and a class called Persons which inherits from both classes.

In the class Grades we have the grades of all students currently in school.

The class furthermore has following attributes: Math, Science, English.

The methods of the class are the following: get_math, get_science, get_english.

In the class Students we are recording the students respective grade and homeroom teacher.

The class has the following attributes: Grade, Homeroom Teacher.

The class furthermore has the following methods: get_grade, get_homeroom_teacher.

We now inherit from the Classes Grades and Students .

The approach is called Multiple inheritance, because we inherit from two classes at the same time.

Multiple inheritance is used when the classes under each other don't relate, whereas multi-level inheritance is used when the classes under each relate.

Example: It wouldn't make much sense for the class Students to inherit from Grades, because we only want to have the same attributes and methods of the two classes in one class called Person.
