Functions

In the first chapter we briefly covered some functions in Python.

The amazing thing is that Python is not only bound to these functions.

Python offers a range of functions where some will be covered in this chapter.

At this point, you are probably asking yourself what a function is.

Well, a function is a mini-program inside your actual program. Nothing more and nothing less.


1.1 Def

With the def function you can define your own function.

The line(s) of code inside your function are the foundation of your function.

The function is only executed when it's called.

You define a function in Python with the keyword def .

With functions you also have the possibility to include argument(s) and parameter(s).

Argument(s) are given to the parameter when the function is called.

Parameter(s) receive the argument(s) which then internally can be used in the function.

After the function has been called the parameter values get destroyed.

The main advantages of using functions are that you avoid repeating yourself in your program and that you don't have to change the code at every point in your program, if you made a mistake or forgot something.


1.2 Return

The return function returns a value to the place where the function has been called.

This function comes in very handy when you are using your defined functions!

You define a return function in Python with the keyword return.

If you don't use the return function in your defined function, then Python will return the value None.

None is used when you don't want to save a real value inside your variable, because every variable needs to hold a value.


1.3 End and Sep

With the end keyword you can make the values in the two print functions fit into a single line.

With the sep keyword you can separate multiple values in your print function with a delimiter.


1.4 Scope

Local variables only exist within functions, whereas global variables exist outside of the function.

Local variables are non-existent after the function where the variables are in has been called.

Global variables are destroyed when the entire program has been executed.

Code in the global scope can't use local variables.

On the other hand the local scope can use and modify global variables.

Local scope code can't access other local scope code.

In Python you can have two variables with the same name, if they both have different scopes.

Many experienced programmers will try to limit the use of global variable, since they can be modified by every function in your program.

If you really have to use global variable inside of function, please put global in front of it.


1.5 Exception

Python will terminate your program, if there's an error.

To avoid this from happening you can use exception handling.

Python will continue with the rest of your program until the execution is done.
